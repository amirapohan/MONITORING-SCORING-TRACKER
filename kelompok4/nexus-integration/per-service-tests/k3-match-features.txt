============================================================
 K3 — TEAM MATCHING : CATATAN FITUR + MOCK PAYLOAD
 Pendamping untuk: k3-match-test.sh
============================================================
Akses lewat gateway: http://localhost  ->  prefix /match (dibuang gateway)
AUTH : header Authorization: Bearer <JWT K1>. K3 validasi token ke K1
       (POST /internal/validate-token). Role K1 talent -> student.
       /teams*, /invites*, /join-requests*, /members* butuh role student.
PERSONA : 4 talent dibuat di K1 -> PO, MEM, MEM2, POOL.
PERIODE : 2026-1   PRODI : "Ilmu Komputer"

[1] HEALTH & ME
    GET /match/health  (tanpa body) -> {"status":"ok"}
    GET /match/me      (tanpa body, header Bearer)
       -> {"user":{...,"student_id":...,"role":"student"}}

[2] POOL
    POST /match/pool  (Bearer) -> status "waiting" (duplikat -> 409)
    {
      "program_studi": "Ilmu Komputer",
      "period": "2026-1",
      "skills": ["python", "react"],
      "sdg_topics": [4, 9],
      "availability": "full-time",
      "notes": "siap remote"
    }
    (wajib: program_studi, period, skills[]; sisanya opsional)

    GET /match/pool?period=2026-1  (Bearer, tanpa body)
       -> {data[],total,page,limit,totalPages}
       query opsional: &program_studi=&sdg_topic=4&status=waiting&page=1&limit=10&skill=python

    DELETE /match/pool/me?period=2026-1  (Bearer, tanpa body)
       -> status "withdrawn"  (hanya bila status masih "waiting")

[3] PROFILE SKILLS (butuh pool entry)
    GET /match/profile/skills?period=2026-1  (Bearer, tanpa body)
       -> {skills, sdg_topics}
    PUT /match/profile/skills  (Bearer) -> "Profil skill berhasil diperbarui"
    {
      "skills": ["python", "go"],
      "sdg_topics": [4],
      "period": "2026-1"
    }

[4] TEAMS  (role student; PO = pembuat tim)
    POST /match/teams  (Bearer PO) -> status "forming"
    {
      "name": "Tim Garuda",
      "period": "2026-1",
      "generation_method": "manual"
    }
    (PRASYARAT: PO sudah join pool periode itu, status waiting)

    GET /match/teams                 (Bearer, tanpa body) -> data[] (forming)
    GET /match/teams/:id             (Bearer, tanpa body) -> data + members[]
       contoh: GET /match/teams/<team-uuid>

    PUT /match/teams/:id/required-skills  (Bearer PO) -> {required_skills}
    {
      "required_skills": ["python", "figma", "postgres"]
    }

[5] RECOMMENDATIONS
    GET /match/recommendations/teams?period=2026-1&limit=10  (Bearer, tanpa body)
       -> {recommendations[]}
    GET /match/recommendations/members?team_id=<team-uuid>&limit=10
       (Bearer PO tim itu, tanpa body) -> {recommendations[]}

[6] INVITES (alur happy-path penuh diuji)
    POST /match/teams/:id/invites  (Bearer PO) -> "pending"
    {
      "invitee_student_id": "<uid K1 si invitee>",
      "message": "gabung tim kami yuk"
    }
    (invitee harus punya pool entry waiting di periode sama)

    PUT /match/invites/:id/respond  (Bearer invitee) -> status "accepted"
    {
      "response": "accepted"
    }
    (nilai valid: "accepted" | "rejected")

[7] JOIN REQUESTS (varian path diuji)
    POST /match/teams/:id/join-requests  (Bearer peminta) -> "pending"
    {
      "message": "izin gabung, skill saya cocok"
    }
    PUT /match/teams/:id/join-requests/:req_id  (Bearer PO) -> 200
    {
      "action": "accepted"
    }
    (varian non-path: POST /match/join-requests {"team_id":"<uuid>","message":"..."} ;
     PUT /match/join-requests/:req/respond {"action":"accepted"})

[8] MEMBER MANAGEMENT (role student)
    DELETE /match/teams/:id/members/:sid  (Bearer PO, tanpa body)
       -> "Member berhasil dikeluarkan"
       contoh: DELETE /match/teams/<team-uuid>/members/<uid-member>
    DELETE /match/teams/:id/members/me    (Bearer member, tanpa body)
       -> "Berhasil keluar dari tim"  (PO tak boleh leave -> po_cannot_leave)

[9] INTERNAL (header x-service-key: <INTERNAL_SERVICE_KEY>, BUKAN JWT)
    GET /match/internal/check-team/:student_id  (tanpa body)
       contoh: GET /match/internal/check-team/<uid-student>
       -> {"success":true,"data":{"student_id":...,"has_team":true|false,...}}
    Tak ada default key -> SKIP kecuali set K3_SERVICE_KEY=...

KNOWN-BUG (cacat nyata di kode K3 — skrip mengonfirmasi status)
------------------------------------------------------------
 (a) GET /match/teams?needs_skill=<x>     -> 500 internal_error
     (getTeamListBySkill tak di-import di teamRoutes.js)
 (b) DELETE /match/members/me             -> 500 internal_error
     (getActiveTeamByMember tak di-import; pakai
      DELETE /match/teams/:id/members/me sebagai gantinya)
 (c) GET /match/teams/:id/join-requests   -> 404 (route tak diimplementasi;
     hanya POST .../join-requests & PUT .../join-requests/:req_id yang ada)
 (d) POST /match/pool -> 500 : INTEGRASI K1<->K3 RUSAK (terdeteksi saat
     run di stack ini). Kolom pool_entries.student_name NOT NULL diisi
     dari payload.student_name token K1, tetapi JWT/validate K1 TIDAK
     mengembalikan field nama -> NULL -> "null value in column
     student_name violates not-null constraint". Akibat: SELURUH rantai
     pool/teams/invites/join-requests/members tak bisa diuji sampai
     kontrak K1 (sertakan nama di payload) atau K3 (kolom nullable /
     ambil nama via /internal/users/:id) diperbaiki. Skrip menandai (d)
     lalu SKIP rantai bergantung pool.
 Catatan: dua konvensi key internal berbeda
   - middleware auth : header x-internal-api-key, env INTERNAL_SERVICE_KEY
     (fallback MOCK_SSO_SECRET, default "your-secret-key-here")
   - /internal/check-team : header x-service-key, env INTERNAL_SERVICE_KEY
     (tanpa fallback/default)
